YouTube OAuth2 (One-Time)

YouTube requires OAuth2. You authenticate once locally — the token is saved and reused (including in the cloud).

Step 1 — Get OAuth2 credentials from Google Cloud Console:

  • Go to console.cloud.google.com/apis/credentials
  • Click + Create Credentials → OAuth client ID
  • Application type: Desktop app
  • Download the JSON → save as client_secrets.json in the project folder
  • Go to OAuth consent screen → add your YouTube email as a Test User
  • Enable YouTube Data API v3 in APIs & Services → Enable APIs

Step 2 — Run auth flow (opens browser once):

bash
# This opens your browser to authorise YouTube access
# Creates youtube_token.pickle (keep this safe!)
python main.py --dry-run
⚠️
Token expiry: YouTube OAuth tokens last ~6 months. When expired, just run python main.py --dry-run locally again and re-upload the token to Secret Manager.

Run with Docker (Local)

Before deploying to GCP, test that everything works inside Docker.

bash — Docker Compose
# Build and run a single dry-run
docker compose up

# Run with a theme hint
docker compose run pipeline python main.py --theme "gym equipment" --dry-run

# Generate 3 videos in sequence
docker compose run pipeline python main.py --count 3 --dry-run

# Print the LangGraph ASCII diagram
docker compose run pipeline python main.py --visualise

# Run the scheduler (posts at 11:00 + 19:00)
docker compose --profile scheduled up scheduler
bash — Build image manually
# Build the production image
docker build -t shortsforge:latest .

# Run it (passing .env as env-file)
docker run --env-file .env \
  -v $(pwd)/output:/app/output \
  -v $(pwd)/youtube_token.pickle:/app/youtube_token.pickle \
  -v $(pwd)/client_secrets.json:/app/client_secrets.json \
  shortsforge:latest
✅
Check output/ — generated videos appear in ./output/videos/ and metadata JSON in ./output/metadata/ on your host machine (mounted via volume).

GCP Project Setup

bash
# Login and set your project
gcloud auth login
gcloud config set project YOUR_PROJECT_ID

# Enable all required APIs in one command
gcloud services enable \
  run.googleapis.com \
  cloudbuild.googleapis.com \
  cloudscheduler.googleapis.com \
  secretmanager.googleapis.com \
  artifactregistry.googleapis.com
bash — Create service account
PROJECT_ID=$(gcloud config get-value project)

# Create the service account
gcloud iam service-accounts create shortsforge-sa \
  --display-name="ShortsForge Service Account"

SA="shortsforge-sa@${PROJECT_ID}.iam.gserviceaccount.com"

# Grant Cloud Run invoker (needed for Scheduler to trigger the job)
gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:${SA}" \
  --role="roles/run.invoker"

Upload Secrets to Secret Manager

Never put API keys in your Docker image. Use Secret Manager — the container fetches them at runtime via entrypoint.sh.

bash
SA="shortsforge-sa@${PROJECT_ID}.iam.gserviceaccount.com"

# Upload API key secrets
echo -n "sk-ant-xxxxx" | gcloud secrets create ANTHROPIC_API_KEY \
  --data-file=- --replication-policy=automatic

echo -n "AIzaSyxxxxx" | gcloud secrets create GOOGLE_API_KEY \
  --data-file=- --replication-policy=automatic

# Upload credential files
gcloud secrets create YOUTUBE_CLIENT_SECRETS \
  --data-file=client_secrets.json --replication-policy=automatic

gcloud secrets create YOUTUBE_TOKEN \
  --data-file=youtube_token.pickle --replication-policy=automatic

# Grant the service account access to all secrets
for SECRET in ANTHROPIC_API_KEY GOOGLE_API_KEY YOUTUBE_CLIENT_SECRETS YOUTUBE_TOKEN; do
  gcloud secrets add-iam-policy-binding $SECRET \
    --member="serviceAccount:${SA}" \
    --role="roles/secretmanager.secretAccessor"
done
🔄
To update a secret later (e.g. refresh YouTube token):
gcloud secrets versions add YOUTUBE_TOKEN --data-file=youtube_token.pickle

Deploy Cloud Run Job

You can deploy with a single command (it builds the Docker image automatically via Cloud Build) — or use the automated script.

Option A — One command:

bash
gcloud run jobs deploy shortsforge \
  --source . \
  --region asia-south1 \         # Mumbai — closest to Bengaluru
  --memory 2Gi \
  --cpu 2 \
  --task-timeout 3600 \          # 1 hour max (Veo takes ~5 min)
  --max-retries 1 \
  --service-account "shortsforge-sa@${PROJECT_ID}.iam.gserviceaccount.com" \
  --set-env-vars "DRY_RUN=false,VIDEO_COUNT=1"

Option B — Automated deploy script (does everything):

bash
# Handles: APIs, service account, secrets, job deploy, scheduler
chmod +x deploy_gcp.sh
./deploy_gcp.sh

Test the job immediately (without waiting for schedule):

bash
gcloud run jobs execute shortsforge --region asia-south1

# Watch live logs in real time
gcloud run jobs executions logs tail shortsforge --region asia-south1

Cloud Scheduler (11AM + 7PM IST)

bash
JOB_URI="https://asia-south1-run.googleapis.com/apis/run.googleapis.com/v1/namespaces/${PROJECT_ID}/jobs/shortsforge:run"
SA="shortsforge-sa@${PROJECT_ID}.iam.gserviceaccount.com"

# 11:00 AM IST (05:30 UTC)
gcloud scheduler jobs create http shortsforge-morning \
  --location asia-south1 \
  --schedule "30 5 * * *" \
  --time-zone "Asia/Kolkata" \
  --uri $JOB_URI \
  --http-method POST \
  --oauth-service-account-email $SA

# 7:00 PM IST (13:30 UTC)
gcloud scheduler jobs create http shortsforge-evening \
  --location asia-south1 \
  --schedule "30 13 * * *" \
  --time-zone "Asia/Kolkata" \
  --uri $JOB_URI \
  --http-method POST \
  --oauth-service-account-email $SA

# Trigger a job immediately (for testing)
gcloud scheduler jobs run shortsforge-morning --location asia-south1
